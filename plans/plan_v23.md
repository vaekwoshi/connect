# 세끌(Secul) 앱 개발 청사진 및 로드맵 (v5.0)

## 📌 개요
본 청사진은 세끌(Secul) 앱의 핵심 특허 청구항을 구현하고, 사용자 경험(UX)을 고도화하기 위한 마스터 로드맵입니다.

---

## 🚀 P0 단계: 특허 청구항 필수 요건 구현 (완료)
가장 핵심이 되는 특허 청구항(결제 데이터 연동, 문턱 돌파 알림, 가상 신고서 제공 등)의 구현 단계입니다.

- [x] **P0-1. 도구-시뮬레이터 연결**: 홈 화면 하단 '도구' 탭에서 연말정산 및 종합소득세 시뮬레이터로 진입하는 내비게이션 연결
- [x] **P0-2. 로컬 푸시 알림 및 스케줄링**: 지출 입력 시 문턱(25%) 돌파 여부에 따른 즉시 푸시 알림 발송 및 7일 후 미입력 시 예약 알림 발송 (`flutter_local_notifications` v22.0.1 적용)
- [x] **P0-3. 맞춤형 세금 신고서 양식 생성**: 시뮬레이터(연말정산/종소세) 결과값을 바탕으로 국세청 양식과 유사한 가상 신고서 UI 제공 (`TaxReportFormScreen`)

---

## 🛠 P1 단계: UI/UX 고도화 및 도구 접근성 강화 (현재 진행 단계)
기본 기능 구현 후, 복잡도를 낮추고 사용성을 높이는 리팩토링 단계입니다.

- [ ] **P1-1. 지출 목표 카드 리팩토링**: 홈 화면의 인라인 입력 폼(신용/체크/현금)을 제거하고, 카드를 터치하면 새 스크린(`ExpenseTargetScreen`)으로 이동하여 이번 달 지출 목표와 상세 내역을 입력/확인하도록 변경 (바텀 시트 금지 규칙 적용)
- [ ] **P1-2. 프리랜서 경비 관리 카드 구현**: 프리랜서 모드 홈 화면에 '경비 증빙 현황' 카드를 배치하여, 다음 세율 구간 하락을 위해 필요한 추가 경비액(절세 넛지) 노출
- [ ] **P1-3. 간편장부 도구 연결**: 하단 '도구' 탭에 `[간편장부]` 진입점을 추가하고 `freelancer_book_screen.dart`로 라우팅 연결

---

## 🌟 P2 단계: 데이터 영속성 및 최적화 (예정)
앱의 완성도를 높이고, 데이터를 안전하게 보관 및 처리하는 단계입니다.

- [ ] **P2-1. 로컬 데이터베이스 연동 강화**: 시뮬레이션 결과 및 지출 내역을 SQLite 기반 DB에 영구 저장 및 불러오기 로직 고도화
- [ ] **P2-2. 배너 및 알림 주기 고도화**: 최상단 광고/배너(7초 롤링, X 클릭 시 72/24시간 숨김, 168시간 후 수정 알림)의 DB 상태 관리 완성
- [ ] **P2-3. UI 애니메이션 및 성능 최적화**: 토스(Toss) 스타일의 미세한 햅틱 피드백 및 스케일 애니메이션 통일 적용
