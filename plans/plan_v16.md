# 종합소득세 진단 기능 구현 계획

사용자님의 질문("종합소득세 진단의 내용과 기능이 어떤게 들어가야하는지 알지?")에 대한 답변이자, 해당 기능을 완벽하게 구현하기 위한 세부 계획서입니다. 사용자 유형별(직장인, 프리랜서, N잡러) 맞춤형 진단 로직과 보안 룰을 엄격하게 적용하여 설계했습니다.

## 1. 사용자 유형별 맞춤형 진단 로직 (Core Features)

### A. 직장인 (경정청구 및 비밀 신고)
- **연말정산 누락분 챙기기**: 연말정산 때 놓친 월세, 기부금, 신용카드 공제를 5월 종소세 신고(경정청구)로 환급받을 수 있도록 시뮬레이션.
- **회사 모르게 비밀 신고 가이드**: 회사에 알리기 싫은 민감한 의료비(난임 시술 등), 특정 단체 기부금 등을 회사 연말정산에서 고의로 누락시키고, 5월 종소세 때 개인이 직접 홈택스로 신고하여 환급받는 '비밀 신고 금액' 계산 기능.

### B. 프리랜서 (3.3% 환급액 및 세액 시뮬레이션)
- **예상 세액 및 환급액 계산**: 기납부세액(수입의 3.3%)과 실제 산출세액을 비교하여 최종적으로 5월에 '환급'을 받을지 '토해낼지' 진단.
- **최적 경비율 자동 판별**: 사용자의 업종코드와 연 수입을 바탕으로 **단순경비율 vs 기준경비율(간편장부)** 중 어느 쪽이 유리한지 자동 비교 추천.
- **프리랜서 특화 공제**: 노란우산공제, 연금저축 납입액 등에 따른 환급액 증가율 시뮬레이션.

### C. N잡러 (소득 합산 및 건보료 방어)
- **합산 소득 과세표준 진단**: 직장 근로소득(세전) + 부업 사업소득을 합산했을 때, 소득세율 구간(15% -> 24% 등)이 점프하는지 경고.
- **건강보험료 폭탄 방어 (매우 중요)**: 근로 외 소득(부업 소득)이 연 2,000만 원을 초과할 경우 발생하는 '소득월액 보험료(건보료 추가 부과)' 도달까지 얼마가 남았는지 게이지바로 시각화하여 경고.
- **유리한 지출 배분**: 신용카드 공제는 근로소득으로, 사업 관련 지출은 사업경비로 완벽히 분리하는 절세 전략 제시.

## 2. 파일 및 데이터베이스 아키텍처

- **데이터베이스 암호화 (Rule 5 적용)**: 종합소득세 진단 시 입력되는 민감 데이터(비밀 신고할 의료비, 부수입 내역 등)는 반드시 암호화(Encryption) 로직을 적용하여 DB에 저장합니다.
- **SQL 쿼리 관리 (Rule 5 적용)**: 종합소득세 진단 관련 DB 쿼리문은 `002_comprehensive_tax_queries.sql` 형태의 번호가 붙은 파일로 분리하여 전용 폴더(`lib/core/data/sql/`)에서 철저히 관리합니다.

### 신규 파일 생성 계획
1. `lib/ui/screens/comprehensive_tax_screen.dart`: 종합소득세 진단 메인 화면 (3가지 유형 분기)
2. `lib/ui/screens/tax_secret_report_screen.dart`: 직장인 비밀 신고 금액 입력용 보안 뷰
3. `lib/core/tax_engine/comprehensive_tax.dart`: 종소세 통합 계산 엔진 (합산 과세, 경비율 비교 로직)
4. `lib/core/data/sql/002_comprehensive_tax_queries.sql`: DB 테이블 생성 및 민감정보 암호화 관련 SQL 모음
