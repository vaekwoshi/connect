# 연말정산 기록부 및 모두보기 메뉴 신설 계획

현재 홈 화면의 '하단 카드'가 연말정산 결과에 의해 덮어씌워지는 구조를 분리하고, 연간 시뮬레이션 결과는 별도의 '기록부'로 보관하는 구조로 개편합니다.

## Proposed Changes

### 1. 데이터베이스 개편 (`lib/core/data/db_helper.dart`)
연말정산 및 종합소득세 진단 결과를 텍스트 형태로 저장하기 위한 `tax_records` 테이블을 신설합니다.
*   **신규 테이블**: `tax_records` (연도, 세금 종류, 세전연봉, 예상 환급액/납부액, 타임스탬프)
*   **추가 메서드**: `insertTaxRecord()`, `getTaxRecords()`

### 2. 홈 화면 탭 네비게이션 활성화 (`lib/ui/screens/home_screen.dart`)
현재 고정되어 있는 바텀 네비게이션(Bottom Navigation)의 상태값을 연동하여, 화면이 실제로 전환되도록 라우터를 구현합니다.
*   `_currentIndex == 0` : 기존 홈 화면 (소비 가이드)
*   `_currentIndex == 4` : **[신규]** 모두보기(전체) 화면

### 3. 모두보기 및 기록부 화면 신설 (`lib/ui/screens/more_screen.dart` 등)
모두보기 탭에 진입 시 진단 기록들을 모아볼 수 있는 메뉴를 제공합니다.
*   **[NEW] `more_screen.dart`**: '연말정산 기록부', '종합소득세 기록부' 메뉴가 나열된 화면
*   **[NEW] `tax_record_list_screen.dart`**: 저장된 기록(텍스트 위주)을 타임라인 형태로 렌더링하는 공통 화면

### 4. 연말정산 진단 화면 수정 (`lib/ui/screens/year_end_tax_screen.dart`)
월간 가이드(프로필)를 덮어쓰지 않고, 기록부 전용 DB에만 저장되도록 로직을 교체합니다.
*   **[MODIFY]**: `_syncAndSaveProfile` 로직 제거
*   **[MODIFY]**: 하단 저장 버튼의 문구를 "결과를 홈 화면에 반영하기" → **"기록부에 저장하기"** 로 변경하고 `insertTaxRecord` 호출

## Verification Plan
1. 연말정산 진단을 수행한 뒤 [기록부에 저장하기] 버튼 클릭 시 정상적으로 스낵바가 뜨며 저장되는지 확인합니다.
2. 하단 탭의 [전체]를 눌러 '모두보기' 화면으로 이동하는지 확인합니다.
3. '연말정산 기록부'를 눌러 저장된 세전연봉과 환급액 결과가 텍스트 형태로 정상 출력되는지 확인합니다.
