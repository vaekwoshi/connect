# 홈 화면 UI/UX 개편: 소득 입력과 시뮬레이션 카드 분리

현재 하나의 카드(Main Menu Card)에 '소득 정보 입력'과 '세금 시뮬레이션(연말정산/종합소득세/모두보기)'이 섞여 있어 사용자 관점에서 목적이 불분명하다는 의견을 적극 수용하여 카드를 분리합니다.

## Proposed Changes

### `lib/ui/screens/home_screen.dart`

**1. [MODIFY] `_buildMainMenuCard` -> `_buildIncomeCard` 로 역할 축소 및 변경**
*   **목적**: 오직 '나의 소득(월급여, 프리랜서 수입 등)'을 입력하고 확인하는 용도로만 사용합니다.
*   **변경점**: 하단에 있던 [연말정산 | 종합소득세 | 모두보기] 텍스트 버튼 영역을 완전히 삭제합니다.

**2. [NEW] `_buildTaxSimulationCard` 신규 위젯 추가**
*   **목적**: 세금 진단과 시뮬레이션을 전담하는 독립된 컴포넌트로 분리하여 직관성을 높입니다.
*   **디자인 구조**:
    *   **타이틀**: "나의 세금 진단" (가칭)
    *   **버튼 배치**: 
        *   상단 Row: `[연말정산 진단]` 버튼과 `[종합소득세 진단]` 버튼을 나란히 큰 사각형 버튼으로 배치 (아이콘 포함).
        *   하단 넓은 버튼: `[과거 시뮬레이션 기록 모두보기]` (클릭 시 `MoreScreen`으로 이동).
*   이로써 '행동(시뮬레이션)'을 유도하는 UI가 훨씬 명확하고 터치하기 쉽게(Premium UX) 개선됩니다.
