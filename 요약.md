## 알림 시스템 — 리마인더 vs 알림 (분리 원칙)

- **리마인더**(개인용) — `lib/core/notifications/reminder.dart` + `custom_reminder_service.dart`. `kind='custom'`은 사용자가 직접 만들고 자유롭게 편집·삭제 가능. `kind='record'`(가계부 기록 넛지)는 최초 1회 시드되며 토글만 가능(삭제 불가). **⚠️ 다음 세션 최우선 작업** — 아래 "남은 작업" 참고.
- **알림**(전체 브로드캐스트) — `lib/core/notifications/system_reminder_catalog.dart`의 `kSystemReminderCatalog`(DB 아닌 코드 정적 정의). on/off는 `reminder_settings` 테이블. `notification_settings_screen.dart`가 `topCategory` 필드 기준으로 섹션을 나눠 표시(세금 일정 / 교통·에너지 / 일자리·행정 / 공제 문턱 / 가계부 & 예산 / 만료 알림 / 기록 넛지).
  - notifId 대역: 1001~1099=세금, 1100번대=교통·에너지, 1200번대=일자리·행정, 2000번대=개인 리마인더/월급/카드(겹치지 않게 유지).
  - **원칙**: 생일·자녀 유무·최근 검진일 등 개인 상황에 의존하는 혜택(기초연금, 아동수당, 국가건강검진 등)은 브로드캐스트에 넣지 않음. 특정 연도 한정 이슈(장병내일적금 가입기한 등)도 "매년 반복" 구조에 안 맞아 제외.

## 가계부 화면 (`expense_calendar_screen.dart`)

`_buildSummaryBar`(달력·목록 탭 상단 수입/지출/잔액) 재설계 완료(2026-07-05) — 라벨-위/값-아래 레이아웃으로 통일(분석 탭 `_summaryCell`과 톤 일치), 수입은 `_incomeColor`(초록), 지출은 `colorDanger`, 잔액은 흑자/적자 색상 + 방향 화살표(↑/↓)로 강조. 달력 그리드(`_buildCalendar`)는 미변경.

### 결제수단 색상
```dart
_incomeColor   = Color(0xFF5CB87A)  // 수익 — soft green
_pmCreditColor = Color(0xFF6B8FD4)  // 신용카드 — steel blue
_pmDebitColor  = Color(0xFFD4A847)  // 체크+현금 — soft amber
_pmOtherColor  = Color(0xFF9E9B96)  // 기타 — warm gray
```

### 핀치 줌 (2단계)
- **1단계**: 기본 7열, 점·바
- **2단계**: 가로 2배 폭 + 가로 스크롤 + 관성 슬라이딩 (`FrictionSimulation`)
- 터치: `Listener` 원시 포인터, `_pointers` 맵으로 핀치 추적

### 인라인 에디터
- 날짜 탭 → 해당 행 아래 에디터 push (`_calScrollCtrl`)
- 수익/지출 분리 입력, 결제수단 3행 × [금액 + 카테고리]
- 카테고리 → AlertDialog 그리드

### 고정 지출 템플릿
- `recurring_templates_screen.dart` — 관리
- `recurring_confirm_screen.dart` — 월별 확인 (대기/확인/건너뜀)
- 달력 상단 미확인 배너 → 확인 화면 진입
