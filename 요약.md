### 홈 (`home_screen.dart`) ✅ 완성
- 상단 회전 배너 — 각 카드 X 아이콘으로 30일간 숨김(`banner_states` 테이블, `saveBannerHideTime`/`_dismissBanner`). 절세팁 카드는 최대 2장만 노출(`_currentTips`, `limit: 2` — N잡러는 큐레이션 팁 7개 전부에 해당돼 과다 노출 방지).
- 현황 패널 (이달 수령액·지출) → 가계부 링크. 예상연봉·지출목표 입력값은 유형별로 완전 독립 저장(`profile_type_values` 테이블, DB v30) — 직장인→N잡러 등 유형 전환 시 이전 유형 값이 그대로 보이지 않음. 프리랜서는 "예상 연봉" 입력 UI 자체가 없음(고정급여 개념 없음, `_isEmployee` 게이트).
- 직장인이 가계부에 근로소득 외 기타수익을 연 누적 300만원 넘게 기록하면 N잡러 전환 여부 확인 다이얼로그 노출(`_checkNjobConversion`) — 확인 시 기존 입력값 그대로 이관.
- 수익·지출 카드는 유형별로 완전 독립: 프리랜서는 헤드라인이 "이번 달 수입 (세후)"로 표시되고 탭하면 `AnimatedSwitcher`로 세전 환산액 페이드 전환(`_showGrossIncome`, 220ms) — 세전 금액은 저장하지 않고 표시 시점에 `net / rate`로만 역산(홈 화면 합계 이중계산 방지). N잡러는 헤드라인이 "이번 달 근로소득 (세전)"으로 근로소득만 표시(과거엔 "수령액"이라는 라벨이 마치 전체 수입인 것처럼 오해를 줬던 버그를 수정, 2026-07-07) — 근로소득 외 수입은 "다른소득" 칩을 탭하면 같은 방식으로 세전 환산(`_otherIncomeChip`, 과거 명칭 "기타 수익"에서 개명). 근로소득('급여')은 간이세액표 기반이라 역산 불가라 탭 대상에서 제외, 근로소득+다른소득을 하나로 합산 표시하지 않음(성격이 달라 오해 소지).
- `신용카드 공제 문턱`(연봉 기반 카드 사용 목표 안내)은 `_isEmployee` 게이트 추가 — 프리랜서 홈 카드에 잘못 노출되던 버그 수정.
- 가계부의 "월급 N일" 칩은 프리랜서 홈에서 숨김(고정 월급날 개념 없음) — N잡러·직장인은 유지.
- 연중 가입 사용자용 1~6월 소급 입력 유도 배너 → `annual_backfill_screen.dart`(결제수단 3분류: 신용카드/체크+현금/기타 총액만 입력, 카테고리 세분화 없음. 이미 기록 있는 달은 건너뜀).
- 리마인더 패널
- 세무도구 패널 (아코디언)
- 우상단: 알림함 벨 배지 + 설정

## 핵심 데이터 모델

### ExpenseItem
```dart
id, date, endDate?,   // endDate 있으면 범위 지출
amount, content,
category,             // 지출 카테고리 (식비/교통/의류 등 10종)
paymentMethod,        // 신용카드 | 체크+현금 | 기타
isBusiness            // (v31) 사업경비 인정 여부 — category와 독립된 별도 플래그
```

### IncomeEntry (v31 확장)
```dart
id, date, amount, incomeType,  // '급여' | '기타' (N잡러) | '사업소득' | '기타소득' (프리랜서)
isWithheld                     // true면 amount가 원천징수 후 실수령(순액) — 세전 금액은 저장 안 하고
                                // 표시 시점에만 역산(사업소득 ÷0.967, 기타소득 ÷0.912)
```
프리랜서는 `isWithheld` 기본값 true(통장엔 원천징수 후 금액이 들어오므로), 직장인·N잡러는 기본 false.
근로소득('급여')은 간이세액표 기반이라 세전 역산이 불가능 — 세전/세후 탭 전환 UI는 근로소득에는 적용하지 않음.

### RecurringTemplate (v25, v31에 isBusiness 추가)
```dart
id, name, amountHint, category, paymentMethod, isBusiness,
dayOfMonth (1~31), sortOrder
```

### RecurringConfirmation (v25)
```
(templateId, year, month) PK
```

## 세무 계산 엔진 (`lib/core/tax_engine/`)

회귀테스트 53건 전원 통과.

- `employee_tax.dart` — 직장인 연말정산 전체 흐름
- `freelancer_tax.dart` — 프리랜서 종소세 (단순/기준경비율·실제 기장). `calculateTaxRange({accumulatedIncome, inputMonths, allowanceCount, occupationCode, ...})` — 단순경비율/기준경비율 두 가정으로 시뮬레이션을 두 번 돌려 min~max 세액 범위를 반환(새 예상치 UI용, 임의의 룩업 테이블 없이 기존 엔진 재사용).

## 알림 시스템 — 리마인더 vs 알림 (분리 원칙)

- **리마인더**(개인용) — `lib/core/notifications/reminder.dart` + `custom_reminder_service.dart`. `kind='custom'`은 사용자가 직접 만들고 자유롭게 편집·삭제 가능. `kind='record'`(월급날+가계부 기록 통합 넛지, 18시)는 최초 1회 시드되며 시각만 편집 가능(제목·주기 고정, 삭제 불가) — `reminder_form_screen.dart`의 `_isFixedKind`가 이 잠금을 담당. 매주 반복은 복수 요일 선택 가능(`Reminder.weekdays`/`effectiveWeekdays`, 요일별 네이티브 알람 개별 예약). 목록은 다음 발화 시각순 정렬. 면허 갱신·여권 만료·전세 만기는 `reminder_list_screen.dart`의 프리셋 칩으로 제목만 미리 채운 커스텀 리마인더 생성 진입점 제공(날짜는 사용자가 직접 입력). `reminder_list_screen.dart`에 시간수정 아이콘(`_editReminderTime`) 추가 — 발화 시각만 별도 다이얼로그로 즉시 수정 가능.
- **기본 제공(이벤트형) 리마인더** — 날짜가 아닌 "사건 발생" 트리거라 `reminders` 테이블에 넣지 않고 `event_reminder_prefs`(DB v28, key/enabled/hour/minute)로 별도 관리. 현재 5종: `budget_alert`(예산 80%·초과, 홈 화면 `_checkBudget`에서 감지 즉시가 아니라 지연 예약 — 16시 이전 감지는 당일 저녁 20시(기본값, 리마인더에서 편집 가능) / 16시 이후는 다음날 9시 고정), `inactivity_nudge`(지출 3일 이상 미기록), `income_inactivity_nudge`(수입 기록 미기록 — 지출 넛지와 별도 토글로 분리, 사용자 요청), `recurring_expense_alert`(고정지출 템플릿 각각의 결제일에 맞춰 `DateTimeComponents.dayOfMonthAndTime`로 매월 반복 예약, `ReminderScheduler.scheduleRecurringExpenses`가 `recurring_templates_screen.dart` 저장 시 및 토글/시각 변경 시 재예약), **`tax_reserve_shortfall`**(신규, 2026-07-07 — 권장 세금·4대보험 적립액 대비 실제 적립(가계부상 "보험/금융" 카테고리 지출) 부족 시 상태 기반 경고, `ReminderScheduler.checkTaxReserveShortfall`가 홈 화면에서 매번 `ReserveEstimate`로 재계산해 판단, 프리랜서·N잡러만 대상). `inactivity_nudge`/`income_inactivity_nudge`는 다음날 9시(기본값, 편집 가능) 지연 예약, 조건 해소 시 자동 취소. `reminder_list_screen.dart`의 "기본 제공" 섹션에서 토글 on/off + 시각 편집(`_editEventTime`, 듀얼 CupertinoPicker) 가능. 5월 종소세 시즌 안내(`scheduleTaxSeason`)는 프리랜서·N잡러일 때 이번 달 적립 현황 텍스트를 동적으로 덧붙임(`_appendReserveStatus`).
- **알림 히스토리 백필** (DB v27) — `zonedSchedule`로 예약된 개인 리마인더·시스템 세무일정은 OS가 백그라운드에서 직접 발화시켜 원래 알림함(`notification_log`)에 전혀 기록되지 않았음. `lib/core/notifications/notification_history.dart`가 앱 실행 시(`ReminderScheduler.scheduleAll`) 마지막 체크포인트(`app_state` 테이블) 이후 실제로 지났어야 할 발화를 역산해 알림함에 채운다(최대 30일 상한). 결제일·카드 알림(`schedulePayday`/`scheduleCardPayments`)은 이번 범위 밖 — 같은 공백 있음, 후속 과제.
- **알림**(전체 브로드캐스트) — `lib/core/notifications/system_reminder_catalog.dart`의 `kSystemReminderCatalog`(DB 아닌 코드 정적 정의). on/off는 `reminder_settings` 테이블. `notification_settings_screen.dart`가 `topCategory` 필드 기준으로 섹션을 나눠 표시(세금 일정 / 교통·에너지 / 일자리·행정 / 공제 문턱 / 가계부 & 예산 / 만료 알림 / 기록 넛지).
  - notifId 대역: 1001~1099=세금, 1100번대=교통·에너지, 1200번대=일자리·행정, 2000번대=개인 리마인더/월급/카드(겹치지 않게 유지).
  - **원칙**: 생일·자녀 유무·최근 검진일 등 개인 상황에 의존하는 혜택(기초연금, 아동수당, 국가건강검진 등)은 브로드캐스트에 넣지 않음. 특정 연도 한정 이슈(장병내일적금 가입기한 등)도 "매년 반복" 구조에 안 맞아 제외.
  - **프리랜서/N잡러 갭 분석(2026-07-06, 미구현)**: VAT·중간예납·5월 종합소득세는 이미 `business:true`로 올바르게 대상 필터링됨. 남은 갭 4가지 — ① 기록 넛지 문구가 "월급날" 전제라 프리랜서에게 어색함, ② 5월 종합소득세 알림에 3.3%/8.8% 원천징수 정산(환급/추가납부) 안내 없음, ③ 지급명세서(원천징수영수증) 발급 확인 알림 없음, ④ 건강보험 지역가입자 전환 안내 없음(4대보험 자진가입 이슈와 연결, 아래 참고). 아직 코드 변경 없음 — 사용자 확인 후 착수.

### 일별 입력 — 풀스크린 전환 (2026-07-07)
과거엔 날짜를 탭하면 달력 그리드 안에 인라인 에디터가 펼쳐지고 `_scrollToEditor()`가 스크롤을 리스트 맨 끝(`maxScrollExtent`)까지 이동시켰는데, 캘린더 위 콘텐츠(적립 카드 등)가 늘어나면서 에디터가 그 달의 마지막 주가 아닐 때 스크롤이 에디터를 지나쳐버려 입력 필드가 화면 밖으로 밀려나는 구조적 버그가 있었다. 앱의 하드 규칙(바텀시트 금지 — fullscreen push / inline expand / AlertDialog만 허용)에서 이미 허용된 "fullscreen push" 패턴으로 전환해 해결:
- 날짜 탭·드래그 다중선택 시 `_toggleSingle`/드래그 종료 핸들러가 `Navigator.push`로 `DayEntryScreen`을 열고, 저장/삭제 완료 후 팝되면 `_load()`로 캘린더 새로고침.
- `DayEntryScreen`은 자체 상태(컨트롤러·소득유형·카테고리 등)를 직접 소유하는 독립 위젯 — 프리필 값은 캘린더 화면이 기존 `_prefillFromDate`/`_prefillForm` 로직으로 계산해 생성자 파라미터로 전달, DB 저장/삭제 로직은 이 화면이 직접 수행(`_save`/`_delete`) 후 `Navigator.pop`.
- 캘린더 자체는 다시 끊김 없는 단일 `buildSection(allRows)` 리스트로 복귀 — `topRows`/`bottomRows` 분할과 `showEditor` 조건부 삽입 제거.
- **Flutter web(release) 렌더링 함정**: `Scaffold.bottomNavigationBar`에 `Padding`/`Container(padding:)`으로 감싼 `Row(children:[Expanded(...)])`를 직접 주면 body·appBar까지 통째로 안 그려지는(예외 없이 완전히 비어 보이는) 버그를 만남 — `SizedBox(height: 고정값)`로 명시적 높이를 줘서 해결. 실제 모바일(sqflite) 빌드에는 영향 없을 가능성이 높지만, 웹 프리뷰로 화면을 검증할 때 유사 증상이 재현되면 이 패턴부터 의심할 것 (상세: 메모리 `flutter-web-bottomnav-expanded-bug`).
