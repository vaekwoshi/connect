# 세끌 계산 엔진 완성도 종합 검토

> 질문: "이 앱에 필요한 계산 엔진이 완성이 잘 될까? 검산이 완벽한가?"
> 답변 요약: **현재 검산은 "완벽하지 않다". 단, 완성은 충분히 가능하며 작업 목록이 명확하다.**
> 근거: 국세청 공식자료 4종(연말정산 책자·금융소득 해설·종소세 매뉴얼·서식 7종) 대조.

---

## 1. 현재 엔진 구조 (있는 것)

| 엔진 | 파일 | 다루는 세금 |
|---|---|---|
| 직장인 | employee_tax.dart | 근로소득공제, 근로세액공제, 신용카드, 의료/교육/기부, 월세, 4대보험 |
| 프리랜서 | freelancer_tax.dart | 단순경비율 추계, 종합소득세, 3.3% 환급예측, 노란우산 |
| N잡러 | combined_tax.dart | 근로+사업 합산, 투잡(2직장) 합산 |
| 세율 | tax_rates.dart | 기본세율 8구간, 인적공제 기본 |

**검증된 정확한 부분 (✅ 신뢰 가능)**
- 기본세율 8구간 — 누진공제액까지 책자와 **완전 일치**
- 근로소득공제 구간율, 근로소득세액공제(55%/30%, 한도)
- 신용카드 소득공제(문턱 25%, 우선순위 배분) — 매우 정교
- 의료비(난임30/본인등15/일반700만), 월세(17/15·1000만) — 2025 최신
- 프리랜서 단순경비율 추계 로직, 3.3% 기납부 환급예측

## 2. 정직한 검산 판단: 왜 "아직" 완벽하지 않은가

### 🔴 A. 실제 계산 버그 (수치가 틀림)

**노란우산공제(소기업·소상공인 공제부금) 한도 오류** — `freelancer_tax.dart:52-60`, `combined_tax.dart:95-102`
| 구간(소득금액) | 앱 한도 | 2025 정답 | 판정 |
|---|---|---|---|
| 4천만 이하 | 500만 | **600만** | ✗ |
| 4천만~6천만 | (없음, 1억까지 300만) | **500만** | ✗ |
| 6천만~1억 | 300만 | **400만** | ✗ |
| 1억 초과 | 200만 | 200만 | ✓ |
→ 구간·금액 모두 구버전. 프리랜서/N잡러 노란우산 가입자 결과가 틀림.
→ `freelancer_tax.dart:54`에 죽은 코드(`50000000` 후 덮어쓰기)도 함께 정리 필요.

**N잡러 월세 세액공제 조건 누락** — `combined_tax.dart:119-122`
- 무주택 세대주·총급여 7천만 이하·종합소득금액 6천만 이하 조건 체크 없이 무조건 적용.
- 직장인 화면(year_end)에는 조건이 있으나 N잡러 엔진엔 없음.

### 🟡 B. 누락된 공제 (계산은 맞지만 항목이 빠져 과소/과대)
앞서 `2025_연말정산_정답지` 검증에서 확인된 것들:
- 연금보험료공제(국민연금)·특별소득공제 보험료(건강·고용) **과표 차감 미연동** ← 거의 전원 해당
- **자녀세액공제**(2025: 첫째25/둘째30/셋째+40) 미반영
- **연금저축·IRP 세액공제**(600/900만, 12/15%) 없음
- 보장성보험료, 혼인세액공제(50만), 표준세액공제(13만) 없음
- 근로소득공제 **2,000만원 상한 캡** 없음
- 의료비 미숙아·선천성이상아 20%, 교육비 본인·취학전·장애인 누락

### 🔴 C. 구조적 미구현 영역 (종합소득세의 핵심 누락)
**금융소득 종합과세 — 사실상 전무**
- 이자·배당 2천만 초과 시 **비교과세**(종합 vs 분리 중 큰 값), **Gross-up 배당가산(10%)**, **배당세액공제** 필요.
- 현재 `calculateFinancialIncomeAmount`는 건보료 산정용(1천만 초과 전액)일 뿐 세액계산 아님.
- → 금융소득 많은 사용자는 계산 불가.

**종합소득 합산 범위 부족**
- 진짜 종합소득세 = 근로+사업+**연금+기타+금융**소득 합산.
- 현재 N잡러 엔진은 근로+사업만. 연금·기타·금융소득 미합산.

## 3. 완성 가능성 판단 — "가능하다"

핵심 뼈대(기본세율·과표·산출세액·근로/사업소득 흐름)가 정확하므로, **나머지는 모듈을 끼워넣는 작업**이다. 새 세금이론이 필요한 게 아니라 공식자료에 수치·산식이 다 있다.

### 완성 로드맵 (우선순위 = 사용자 영향 × 구현난이도)

**Tier 1 — 정확도 직결, 난이도 낮음 (먼저)**
1. 노란우산 한도 수정(600/500/400/200) — 버그픽스, 즉시
2. 4대보험값 과표 차감 연동 — 엔진 이미 있음, 연결만
3. 자녀세액공제 모듈(인적공제 자동) — 단순 테이블
4. 근로소득공제 2000만 캡 추가 — 한 줄

**Tier 2 — 절세 핵심, 난이도 중**
5. 연금저축·IRP 세액공제 전용 계산기(이미 "준비중")
6. 보장성보험료·혼인·표준세액공제
7. N잡러 월세 조건 가드 추가

**Tier 3 — 고급 사용자, 난이도 높음**
8. 금융소득 종합과세(비교과세+Gross-up+배당세액공제) 모듈
9. 연금·기타소득 종합합산 확장

→ Tier 1·2만 완료해도 직장인·프리랜서·N잡러 일반 사용자 검산은 **신뢰 가능 수준** 도달.
→ Tier 3는 타겟(고액 금융소득자) 포함 여부를 결정해야 함. 일반 앱이면 "종합과세 대상입니다, 세무사 상담" 안내로 갈음 가능.

## 4. 검산 신뢰성 확보 방안 (검증 자동화)

공식자료에 **숫자 예시(정답)가 있다** → 회귀 테스트로 박제 가능:
- 연말정산 책자 p212~ 종합사례(직장인 전체 계산 예시)
- 금융소득 해설 p137 사례(산출세액 18,136,000 / 배당세액공제 3,000,000) — 본 JSON에 박제됨
- 프리랜서/N잡러는 단순경비율 + 기본세율로 손계산 대조 가능

이 예시들을 `test/` 케이스로 만들면 "검산 완벽"을 **객관적으로 증명**할 수 있다.
현재는 `scratch/test_calc.dart`만 있고 공식 정답 대조 테스트가 없음.

## 5. 결론

- **"지금 검산이 완벽한가?" → 아니오.** 노란우산 버그 + 누락 공제 다수 + 금융소득 종합과세 부재.
- **"완성할 수 있는가?" → 예.** 뼈대가 정확하고, 모든 수치·산식을 공식자료로 확보함(인터넷 미사용).
- **권고 순서**: ① 노란우산 버그 즉시 수정 → ② Tier 1 연동(4대보험·자녀·캡) → ③ 공식사례 회귀테스트 구축으로 "검산 완벽" 입증 → ④ Tier 2 절세항목 → ⑤ Tier 3는 타겟 결정 후.

> 정직한 한 줄: **검산은 곧 완벽해질 수 있지만, 지금 "완벽하다"고 말하면 거짓이다.** Tier 1 + 회귀테스트까지 끝내면 일반 사용자 대상으로 "완벽하다"고 말할 수 있다.
