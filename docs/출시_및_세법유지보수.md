## B-2. 실기기 테스트 런북

### ⚠️ 0단계 — JDK 복구 (현재 안드로이드 빌드 블로커)
이 PC는 Android SDK는 있으나 **정상 JDK가 없음**(Android Studio JBR의 `jvm.cfg` 손상). 둘 중 하나:
- **(권장) Android Studio 재설치/복구** → 번들 JBR 정상화 후:
  `flutter config --jdk-dir="C:\Program Files\Android\Android Studio\jbr"`
- **또는 JDK 17 설치**(Temurin/Microsoft OpenJDK) → `flutter config --jdk-dir="<JDK17 경로>"`
- 확인: `flutter doctor` 의 Android toolchain 이 `[√]` 가 되면 OK.

### 안드로이드 (Windows에서 가능)
1. 기기: 설정 → 개발자 옵션 → **USB 디버깅** 켜고 USB 연결(또는 무선 디버깅).
2. `flutter devices` 에 기기가 보이는지 확인.
3. **디버그 실행(서명 불필요)**: `flutter run -d <기기id>` — 핫리로드로 바로 테스트.
4. 설치형 APK: `flutter build apk --debug` → `build/app/outputs/flutter-apk/app-debug.apk`.
5. 릴리스 점검(서명 필요): `key.properties` 채운 뒤 `flutter build apk --release`.

### iOS (macOS + Xcode 필요 — 이 PC에선 불가)
1. Mac에서 `flutter pub get` → `open ios/Runner.xcworkspace`.
2. Xcode → Signing & Capabilities → 본인 **Team** 지정, Bundle ID를 고유값으로.
3. 기기 연결 후 `flutter run -d <iphone>` 또는 Xcode Run.

### ⚠️ 출시 전 필수 변경
- **`applicationId`/Bundle ID**: 현재 `com.example.secul` 은 스토어 등록 불가. 본인 소유 역도메인으로 변경
  (`android/app/build.gradle.kts` 의 `applicationId`, iOS Bundle ID).

### 기기에서 꼭 확인할 항목 (웹에서 검증 못 한 것)
- **시즌 알림**: 유형 전환·알림 토글 후 예약되는지(웹 미지원이라 기기에서만).
- **온디바이스 DB**: sqflite 저장/로드, 데이터 파기 동작.
- **PDF 가져오기**: file_picker + PDF 파싱(연말정산/사업소득 기록).
- **아이콘·스플래시**: 런처 아이콘(세끌)·스플래시 표시.
